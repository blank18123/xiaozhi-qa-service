## 三、各接口测试详情

### 3.1 文本问答 — ✅ 正常

**路由**：`POST /api/v1/ask`

**请求格式**（JSON）：

```json
{
    "query": "你好，你是谁",
    "mode": "text",
    "session_id": null
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `query` | string | 是 | 用户输入文本 |
| `mode` | string | 是 | 固定 `"text"` |
| `session_id` | string | 否 | 会话ID，不传自动生成 |
| `role_prompt` | string | 否 | 自定义系统提示词 |

**响应**：`text/event-stream`（SSE 流式）

```
data: {"event":"session_id","data":"abc123..."}
data: {"event":"token","data":"你好"}
data: {"event":"token","data":"！我是小智..."}
data: {"event":"done","data":"abc123..."}
```

| SSE 事件 | 含义 |
|----------|------|
| `session_id` | 返回会话ID（首个事件） |
| `token` | LLM 流式输出的文本片段 |
| `tool_call` | 触发工具调用（当前未启用） |
| `done` | 问答结束 |
| `error` | 错误信息 |

**curl 示例**：

```bash
curl -X POST http://localhost:8011/api/v1/ask \
  -H "Content-Type: application/json" \
  -d '{"query":"你好，你是谁","mode":"text"}'
```

### 3.2 语音问答 — ✅ 正常

**路由**：`POST /api/v1/ask/audio`

**请求格式**（multipart/form-data）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `file` | file | 是 | 音频文件（WAV, 16kHz 单声道） |
| `session_id` | string | 否 | 会话ID |

**响应**：`audio/wav`

**处理链路**：

```
音频文件 → VoskASR 识别 → 文字 → DeepSeek LLM → 回复文字 → EdgeTTS → 音频 WAV
```

**curl 示例**：

```bash
curl -X POST http://localhost:8011/api/v1/ask/audio \
  -F "file=@test.wav" \
  -F "session_id=abc" \
  -o answer.wav
```

### 3.3 多轮对话 — ✅ 正常

**路由**：`POST /api/v1/ask`（同上，传入相同 `session_id` 即可保持上下文）

```bash
# 第一轮：建立上下文
curl -X POST http://localhost:8011/api/v1/ask \
  -H "Content-Type: application/json" \
  -d '{"query":"我叫小明，今年25岁","mode":"text","session_id":"my-session"}'

# 第二轮：验证记忆
curl -X POST http://localhost:8011/api/v1/ask \
  -H "Content-Type: application/json" \
  -d '{"query":"我叫什么名字？多大？","mode":"text","session_id":"my-session"}'
```
